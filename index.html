Task (2) Data Analysis Report

1. LLM SQL Error Analysis

In this task, we test the ability of LLM to generate SQL for telecom customer churn dataset.

1. The first SQL error: LLM called non-existing database fields, leading to column not found error.

2. The second SQL error: wrong field name and structural confusion of the telecom dataset.

3. The third SQL error: logical omission and incorrect screening conditions.

2. Kaplan-Meier Survival Analysis

We use Kaplan-Meier method to analyze customer churn factors.

Key features such as online security and technical support show significant impact on user retention.

The contract feature did not show significant difference in survival curve comparison, so it was not ranked.

3. Overall Conclusion

Large language models are prone to field hallucination and logical errors when generating SQL. Manual checking and correction are necessary for data analysis work.
